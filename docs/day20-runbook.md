# Day 20 Runbook

## Topic
Terraform local IaC and state management.

## Objective
- Use Terraform to generate config artifacts and manage them via state.
- Use a local backend to store state file.
- Create files for host mappings and app config using variables.
- Include variables and outputs.
- Ensure destroy cleans up all artifacts.

## Evidence Required
1. infra/terraform/ committed to GitHub.
2. plan/apply/destroy outputs captured.
3. Variables documented in README.

- User: webadmin

- Terraform folder: ~/project/infra/terraform

  sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg

## Step 6: Create main.tf

```bash
cat > main.tf <<'ENDOFFILE'
terraform {
  required_version = ">= 1.0"

  required_providers {
    local = {
      source  = "hashicorp/local"
      version = "~> 2.0"
    }
  }
}

# Create output directory placeholder
resource "local_file" "output_dir_placeholder" {
  content  = "# Terraform managed output directory\n"
  filename = "${var.output_dir}/.terraform-managed"
}

# Create hosts mapping file
resource "local_file" "hosts_mapping" {
  content  = <<-HOSTS
    # Terraform managed hosts mapping
    # Generated by: ${var.app_name} - ${var.environment}
    # Do not edit manually

    ${var.minikube_ip}  ${var.host_entry}
  HOSTS
  filename = "${var.output_dir}/hosts-mapping.txt"
}

# Create app config file
resource "local_file" "app_config" {
  content  = <<-CONFIG
    # App Configuration
    # Generated by Terraform for ${var.environment} environment
    # App: ${var.app_name}

    APP_NAME=${var.app_name}
    APP_ENV=${var.environment}
    APP_PORT=${var.app_port}

    # Database config
    DB_HOST=${var.db_host}
    DB_NAME=${var.db_name}
    DB_USER=${var.db_user}

    # Network config
    APP_HOST=${var.host_entry}
    APP_URL=http://${var.host_entry}
  CONFIG
  filename = "${var.output_dir}/app.env"
}

# Create deployment info file
resource "local_file" "deployment_info" {
  content  = <<-INFO
    # Deployment Info
    # Managed by Terraform

    RELEASE_NAME=${var.app_name}-${var.environment}
    NAMESPACE=${var.environment}
    IMAGE_NAME=${var.app_name}
    MINIKUBE_IP=${var.minikube_ip}
    HOST_ENTRY=${var.host_entry}
    APP_URL=http://${var.host_entry}
  INFO
  filename = "${var.output_dir}/deployment-info.txt"
}
ENDOFFILE
```

## Step 9: Create terraform-README.md

```bash
cat > TERRAFORM-README.md <<'ENDOFFILE'
# Day 20 Terraform - Local IaC and State Management

## What this does
Uses Terraform with a local backend to generate config artifacts
and manage them via state. Creates files for host mappings and
app config. Destroy cleans up all artifacts automatically.

## Variables

| Variable    | Type   | Default               | Description                          |
|-------------|--------|-----------------------|--------------------------------------|
| app_name    | string | training-site         | Name of the application              |
| environment | string | dev                   | Deployment environment               |
| app_port    | number | 80                    | Port the application listens on      |
| minikube_ip | string | 192.168.49.2          | Minikube IP for hosts mapping        |
| host_entry  | string | mysite.sj             | Hostname for the app                 |
| db_host     | string | localhost             | Database host                        |
| db_name     | string | trainingdb            | Database name                        |
| db_user     | string | trainingapp           | Database username                    |
| output_dir  | string | /tmp/terraform-output | Output directory for generated files |

## Outputs

| Output               | Description                          |
|----------------------|--------------------------------------|
| app_name             | Application name                     |
| environment          | Deployment environment               |
| hosts_mapping_file   | Path to generated hosts mapping file |
| app_config_file      | Path to generated app config file    |
| deployment_info_file | Path to generated deployment info    |
| app_url              | Application URL                      |

## Commands

  terraform init                          Initialize providers and backend
  terraform plan -var-file="dev.tfvars"   Preview what will be created
  terraform apply -var-file="dev.tfvars"  Create all resources
  terraform destroy -var-file="dev.tfvars" Delete all created resources

## Files created by Terraform

  /tmp/terraform-output/.terraform-managed   Directory marker
  /tmp/terraform-output/hosts-mapping.txt    Hosts entry for mysite.sj
  /tmp/terraform-output/app.env              Application environment config
  /tmp/terraform-output/deployment-info.txt  Deployment details

## State management

  terraform.tfstate        Local state file tracking all resources
  terraform.tfstate.backup Backup of previous state (auto-created)
  Both files are gitignored and never committed to repo.
ENDOFFILE
```

## Step 10: Run Terraform commands and capture output

### Initialize

```bash
cd ~/project/infra/terraform
terraform init
```


### Plan

```bash
terraform plan -var-file="dev.tfvars" \
  2>&1 | tee ~/project/docs/day20-terraform-plan.txt
```


### Apply

```bash
terraform apply -var-file="dev.tfvars" -auto-approve \
  2>&1 | tee ~/project/docs/day20-terraform-apply.txt
```


### Verify files were created

```bash
ls -la /tmp/terraform-output/
cat /tmp/terraform-output/hosts-mapping.txt
cat /tmp/terraform-output/app.env
cat /tmp/terraform-output/deployment-info.txt
```

### Check outputs printed by Terraform

```bash
terraform output
```

Expected output:
  app_config_file      = "/tmp/terraform-output/app.env"
  app_name             = "training-site"
  app_url              = "http://mysite.sj"
  deployment_info_file = "/tmp/terraform-output/deployment-info.txt"
  environment          = "dev"
  hosts_mapping_file   = "/tmp/terraform-output/hosts-mapping.txt"

### Destroy

```bash
terraform destroy -var-file="dev.tfvars" -auto-approve \
  2>&1 | tee ~/project/docs/day20-terraform-destroy.txt
```


Expected output:
  Destroy complete! Resources: 4 destroyed.

### Verify files are gone

```bash
ls /tmp/terraform-output/
```

Output should be empty or directory not found.
